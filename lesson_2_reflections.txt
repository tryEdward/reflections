How is staging area different from the working directory and the repository? What value do you think it offers?

	The staging area is like a temporary holder for the files to be put from the working directory to a new repository. It gives users a buffer zone to have more control over what to commit.

What happens when you initialize a repository? Why do you need to do it?

	It creates a new repository ready to be committed, but you will have to add files to the staging area first to commit the files in the staging area which will be put in the repository initialized.

How can you use the staging area to make sure you have one commit per logical change?

	You could custom add changes from which files to the next new repository.

What are some situations when branches would be helpful in keeping your history organized? How would branches help?

	When you want to try out experimental features while keeping a stable version for public release, branches allows experiments to be committed without worrying about messing up the entire project.
